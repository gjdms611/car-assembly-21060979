# CLAUDE.md

이 파일은 이 저장소에서 작업할 때 Claude Code(claude.ai/code)에게 제공하는 가이드입니다.

## 명령어

Python 프로젝트

- 실행: `python assembly.py`
- 대화형 종료: 아무 프롬프트에서나 `exit` 입력

## 비즈니스 규칙 (PRD.md 기준)

차량 제조 순서: 차량 타입 선택 -> 부품(엔진, 제동장치, 조향장치) 각각 선택 -> 완성된 차량 테스트(선택 부품이 타입에 사용 가능한지 검사)

호환성 제약:
- Bosch 제동장치는 Bosch 조향장치와만 호환 (타사 조향장치 불가)
- Continental 제동장치는 Sedan에 사용 불가
- Toyota 엔진은 SUV에 사용 불가
- WIA 엔진은 Truck에 사용 불가
- Mando 제동장치는 Truck에 사용 불가

차량 타입은 향후 추가될 수 있음 (현재 Sedan/SUV/Truck) -> 새 타입 추가가 쉬운 구조로 설계.

## 리팩토링 방향 (PRD.md 기준)

기존 시스템 문제점: 절차지향 코드로 유지보수 어려움, 안전하지 않은 문법 사용, 확장성 미고려, 유닛테스트 없음.

리팩토링 제약사항:
- **동작 유지**: 콘솔 메뉴 텍스트, 한글 메시지, 입력 방식(숫자 입력, `0` 뒤로가기, `exit` 종료), 호환성 판정 결과는 리팩토링 전후 완전히 동일해야 함
- **테스트**: pytest 기반 유닛테스트 도입, 호환성 규칙과 상태 전이에 대한 커버리지 확보
- **확장성**: 새 타입/부품/규칙 추가 시 코드 수정은 허용되나, 규칙이 한 곳에서만 관리되어 여러 파일/함수를 중복 수정할 필요는 없어야 함
- **Out of scope**: 신규 차량 타입/부품 실제 추가, UI/UX 변경, 다국어화는 이번 리팩토링 범위 밖
